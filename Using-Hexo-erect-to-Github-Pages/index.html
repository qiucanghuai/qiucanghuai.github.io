使用Hexo架设到Github Pages全指南

发表于 2015-10-18   |   分类于 Git应用   |  

Hexo介绍

快速、简单且功能强大的 Node.js 博客框架。
A fast, simple & powerful blog framework, powered by Node.js.

特性

  • 风一般的速度
    Hexo基于Node.js，支持多进程，几百篇文章也可以秒生成。
  • 流畅的撰写
    支持GitHub Flavored Markdown和所有Octopress的插件。
  • 扩展性
    Hexo支持EJS、Swig和Stylus。通过插件支持Haml、Jade和Less.

那么静态网站生成器有哪些？

  • Ruby

    • Jekyll （github 默认pages 引擎）
    • Octopress （兼容jekyll）

  • Python

    • Hyde Jekyll的Python语言实现版本
    • Cyrax 使用Jinja2模板引擎的生成器

  • PHP

    • Phrozn PHP语言实现的静态网站
    • List item

  • JS

    • Hexo

Hexo安装

创建hexo文件夹

$ mkdir blog

安装

npm install hexo -g

升级

npm update hexo -g

初始化

hexo init

安装依赖包

npm install

生成静态文件

hexo generate

本地浏览

hexo server

配置Hexo到github

配置_config.yml文件

文件夹根目录 _config.yml打开修改

deploy:
type: git     //改成git
repository: https://github.com/yourdomain/yourdomain.github.io    #该提交方式需要密码
branch: master

【点击查看更多配置】

创建仓库

  1. 右上角-New repository
  2. Repository name —> input your domain
  3. 点击 Create repository

生成Github pages

  1. 点击右边菜单栏 -> Settings
  2. 在跳转到的页面 Update your site 对应处点击Automatic page generator按钮
  3. 点击继续，选择主题，点击Publish page发布。

使用 ssh-key 免去部署密码

  1. 初始化提交者信息
    git config –global user.email “user@qq.com”
    git config –global user.name “name”
  2. 生成一个SSH keys（参考输入任意一条）
    ssh-keygen -t rsa -C "your_email@example.com"
    ssh-keygen -t rsa -C "your_email@example.COM" -f ~/.ssh/gitcafe #更改密匙名称
  3. 添加SSH keys
    打开github.com 添加id_rsa.pub

  4. 加入SSH Agent
    ssh-agent -s
    出现Agent pid XXXX这样的提示之后，输入：
    ssh-add ~/.ssh/id_rsa
    再输入eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_rsa

    这样，你成功的在本地生成了一个可用的SSH key。

  5. 将SSH Key添加到Github中
    添加之后输入
    ssh -T git@github.com
    提示：Hi username! You’ve successfully authenticated, but GitHub does not
    provide shell access.#表示成功

  6. _congif提交设置
    deploy repository:git@github.com:Xuanwo/xuanwo.github.io.git
  7. 更详细查看
    Generating SSH keys
    如何安装和设置 Git
    如何同时使用多个公秘钥

上传github

sudo npm install hexo-deployer-git --save
hexo deploy

Hexo更换皮肤、安装插件

安装主题

Hexo提供了很多主题，【点击查看更多主题】
推荐使用主题:NexT 亦是本博客使用的主题
修改_config.yml theme 为下载主题名称

安装多说评论

  1. 登陆多说，登录后在首页选择 “我要安装”。
  2. 创建站点，填写站点相关信息。多说域名 这一栏填写的即是你的 duoshuo_shortname

  3. 创建站点完成后在 站点配置文件_config.yml 中新增 duoshuo_shortname 字段，值设置成上一步中的值。

    更多点击【Hexo使用多说教程】

安装百度统计

  1. 登录 百度统计，定位到站点的代码获取页面
  2. 复制 hm.js? 后面那串统计脚本 id
  3. 编辑 站点配置文件_config.yml，新增字段 baidu_analytics 字段，值设置成你的百度统计脚本 id。

新建404页面

在根目录Hexo/source目录下创建 404.html。

---
layout: default
---
<html>
    <head>
         <meta charset="UTF-8" />
         <title>404</title>                                                                                                                                        
    </head>
    <body>
         <script type="text/javascript" src="http://www.qq.com/404/search_children.js" charset="utf-8"></script>
    </body>
</html>

也可以简化为这一行：

<script type="text/javascript" src="http://www.qq.com/404/search_children.js" charset="utf-8"></script>

导航栏添加单页链接

hexo new page "about"
到source/about/index.md编辑内容。
在themes/light/_config.yml中，添加如下：
menu:
  关于: /about

sitemap插件

  1. 安装
    npm install hexo-generator-sitemap

  2. 启用

修改Hexo\_config.yml
# Extensions
Plugins:
- hexo-generator-sitemap

#sitemap
sitemap:
  path: sitemap.xml

  1. 使用方法

    1. 访问 http://localhost:4000/sitemap.xml，即可看到站点地图。

    2. 修改themes//_config.yml，menu节点下添加下面的内容 

      menu:
Home: /
Archives: /archives
Rss: /atom.xml
Sitemap: /sitemap.xml

添加favicon.ico

将favicon.ico图标文件放在source目录下。

生成post时默认生成categories配置项

在scaffolds/post.md中，添加一行categories:
categories:
- test1
- test2

购买域名、设置DNS

进入source文件夹，创建名为CNAME的文件，编辑：
YourDomain.com

写文章

新建文章

hexo new [layout] “postName” #新建文章

其中layout是可选参数，默认值为post。有哪些layout呢，请到scaffolds目录下查看，这些文件名称就是layout名称。当然你可以添加自己的layout，方法就是添加一个文件即可，同时你也可以编辑现有的layout，比如post的layout默认是hexo\scaffolds\post.md

文章中插入图片

使用markdown写文章，插入图片的格式为![图片名称](链接地址)，这里要说的是链接地址怎么写。对于hexo，有两种方式：

  1. 使用本地路径：在hexo/source目录下新建一个img文件夹，将图片放入该文件夹下，插入图片时链接即为/img/图片名称。
  2. 使用微博图床，地址http://weibotuchuang.sinaapp.com/，将图片拖入区域中，会生成图片的URL，这就是链接地址。
  3. 使用七牛

总结-常用命令

hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本

hexo现在支持更加简单的命令格式了，比如：
hexo g == hexo generate
hexo d == hexo deploy
hexo s == hexo server
hexo n == hexo new

一般出错可以试试用这几种方式解决：

  1. 检查配置文件_config.yml
  2. 更新ssh
  3. 更新npm，hexo版本
  4. 删除目录下.git文件夹（隐藏）再重新git init
  5. 删除github上的repository，重新建立再deploy，之前的commit记录仍会保存，只要不重新init
